Part I
Why Automated Raman
Identification?

Raman spectroscopy in one paragraph

Inelastic scattering of monochromatic light produces a wavenumber-shifted spectrum whose peaks correspond to the vibrational modes of chemical bonds. The peak positions, widths, and relative heights together act as a fingerprint for a given mineral phase.

  • Non-destructive, no sample preparation
  • Works on solids, liquids, and powders
  • Field-portable instruments now exist
  • Public reference libraries (RRUFF) cover >4000 mineral species

Application domains

  • Geology & planetary science (Mars Perseverance/SHERLOC)
  • Cultural-heritage pigment analysis
  • Forensic mineralogy
  • Process control in mining and ceramics
  • Educational laboratories
Bottleneck
Identification is mostly manual: an expert compares peak lists by eye. A reproducible, deterministic pipeline is missing in many low-resource labs.

The need this project addresses

Existing tools

  • Vendor software bundled with spectrometers — closed, expensive, often offline-only
  • RamanSPy / matplotlib scripts — research code, not user-facing
  • RRUFF web search — only covers a single specimen at a time, no preprocessing

What we want

  • A web app: drag-drop a CSV, see a result in seconds
  • Pure-mineral identification and mixture decomposition
  • Two complementary identifiers — fast spectral matching and a learned model — and an automatic dispatch between them
  • Open source, runs on a laptop with a single GPU
Frame this as filling a tooling gap, not as proposing a novel algorithm. The intellectual contribution is in the integration and the engineering, not in a new model architecture.
Part II
Problem Formulation

Formal statement

Given a Raman spectrum {(νi, Ii)}i=1..N of an unknown sample, output a ranked list of candidate minerals (mk, ck) with confidence ck ∈ [0,1], plus, if applicable, a fractional decomposition into known phases.

Constraints we impose

  • Wavenumber range fixed to a canonical grid: 100 – 4000 cm⁻¹ at 2 cm⁻¹ step → 1951 points
  • Reference library: 25 curated synthetic Lorentzian profiles + ~2 456 RRUFF averages = 2 481 unique minerals
  • Latency target: end-to-end response < 1 s on a laptop GPU (RTX 3050)
  • Identification produces a top-K list (K = 5 here) with calibrated relative confidences
Be explicit that the canonical grid is a simplification — real spectrometers report arbitrary, non-uniform sampling. We flag the consequence later in the limitations.

Part IV
Methodology

Preprocessing — order matters

Pipeline

  1. Linear interpolation onto the canonical 2 cm⁻¹ grid first
  2. Cosmic-ray removal — neighbour-ratio test, local 5-point window
  3. Savitzky–Golay smoothing — window = 5, polyorder = 3 (≈ 10 cm⁻¹ physical)
  4. Asymmetric Least Squares baseline (Eilers & Boelens, 2005), λ = 10⁵, p = 0.01
  5. Negative-clip and L2 normalisation

Why interpolate first

Smoothing performed on the raw input grid couples the S-G window length to the spectrometer's native sampling. A CSV at 4 cm⁻¹ would receive a 44 cm⁻¹ window; a high-resolution RRUFF scan at 0.5 cm⁻¹ would receive 5 cm⁻¹. Doing all of step 2 onward on the canonical grid guarantees every spectrum sees an identical filter response.

Why neighbour-ratio for cosmic rays? Global modified-Z-score flags genuine narrow Raman peaks (e.g., quartz 464 cm⁻¹) as spikes. A 5-point local test correctly distinguishes broad Raman peaks (elevated neighbours) from cosmic rays (isolated spikes).
Emphasise: interpolation order is not a trivial "engineering choice" — it changes measured CNN accuracy by tens of percentage points on out-of-distribution queries.

Reference database — keep both ideal and real

For the 25 minerals where we author a synthetic Lorentzian profile from published peak tables, we keep both the synthetic row and the RRUFF average as separate matrix rows. The matcher dedupes by display name afterwards.

SourceStrengthWeakness
Synthetic (n = 25)Clean, canonical wavenumbers; perfect anchor for low-noise queriesCoverage limited to common minerals
RRUFF averaged (n = 2 456)Real instrumental bandshape; broad species coverageIncludes "unknown" specimens, mis-labelled samples, baseline residuals

An earlier iteration kept only one of the two for collisions and consistently scored ~79 % against the wrong mineral whenever the two distributions disagreed in shape. Carrying both rows costs 25 extra entries in a 2 481-row matrix.

Design principle
Coverage and canonicity are different requirements; do not collapse them to a single representative.

Spectral matcher — combined metric

For each reference row ri we compute

s(q, r) = ½ · ⟨q̂, r̂⟩ + ½ · ⟨D²[q]̂, D²[r]̂⟩

where q̂ is the L2-normalised query intensity, D² is the Savitzky–Golay 2nd derivative, and the hat denotes mean-centred unit-norm.

L2 cosine alone

Sensitive to broad spectral shape; over-rewards minerals whose RRUFF average drifts in the same direction as the query.

2nd-derivative Pearson alone

Emphasises peak position and sharpness; ties unrelated minerals that happen to have a sharp peak near a query peak.

The 50-50 blend lets each metric veto the other. Empirically this cleanly separates Quartz from spectral aliases such as Sodalite and Amicite.

Weights (½, ½) were chosen by grid search over a held-out RRUFF panel; gains plateaued around equal weighting.

Training-data construction

851 mineral classes pass the filter min_spectra ≥ 2. For each class, the curated synthetic profile is appended to the RRUFF samples so the network observes both clean Lorentzians and real instrumental bandshapes.

Per-variant augmentation pipeline

  1. Gaussian intensity noise σ ∈ [0.002, 0.025]
  2. Multiplicative laser-power jitter ∈ [0.75, 1.25]
  3. Wavenumber shift ±10 cm⁻¹ (calibration drift)
  4. Polynomial baseline residue (degree 1–3, p = 0.30, amp = 0.01 max)
  5. Optional moving-average broadening (p = 0.40, w ∈ {2..4})
  6. Optional spectral dropout (p = 0.20, ≤ 4 points to zero)
  7. Clip to non-negative + L2 renormalise
Lesson learned Adding the synthetic profile to every common-mineral class (not just the ones RRUFF didn't cover) was the single change that fixed the demo path.

Mixture decomposition

Given a query q ∈ ℝ¹⁹⁵¹, find non-negative weights x ≥ 0 minimising

||Aᵀ x − q||₂

where A is a candidate-row matrix.

Candidate-set construction

  • Always include the 25 synthetic rows — well-separated, well-conditioned basis.
  • Add a RRUFF row only if its combined-metric similarity strictly exceeds the best synthetic candidate's. This signals "the query lies outside the synthetic span" and prevents NNLS from spreading credit across spectral aliases of common minerals.
Why not NNLS over the whole library? 1 951 equations versus ≈ 2 500 unknowns is severely underdetermined. NNLS finds some sparse non-negative explanation, but on pure Calcite it returned 0 % Calcite in a top-5 of obscure carbonates.

Auto-mode dispatcher

Every available identifier is run; the dispatcher returns the candidate with the highest top-1 confidence, subject to:

  • Spectral matching is always run as a safety net (cheap, deterministic).
  • A learned model (CNN, RF) must produce top-1 ≥ 0.55 to enter the comparison; below that its softmax is uncalibrated and prone to confidently-wrong out-of-distribution predictions.
  • Ties broken in favour of the lower-variance method (matcher).
Why this matters A naive "always prefer CNN" policy made the system answer Iowaite for Quartz on demo samples — a result that survived training accuracy of 98.6 % because Iowaite is in-distribution and the synthetic Quartz query was not.

Part VI
Where the system
does not meet the bar

Honest limitations · 1 / 4 — Distribution shift

Symptom The CNN was originally trained only on RRUFF spectra. On synthetic-style demo queries it answered Iowaite for Quartz at 41 % confidence — high enough to look authoritative.

Why it happens

  • RRUFF measurements carry instrument-specific noise, baselines, and minor satellite peaks the network learns to exploit.
  • Synthetic Lorentzian inputs lack those features → activations diverge → softmax spreads probability over a near-arbitrary class.
  • Validation accuracy (89 %) was computed on the same augmented distribution, so it never warned us.

Mitigations applied — and their ceiling

  • Mix synthetic profiles into every training class. Lifts CNN top-1 on synthetic CSVs from ~0 % to 86 %.
  • Auto-mode confidence threshold — the safety net.
  • Open problem: we still cannot certify a CNN prediction on a never-seen instrument's noise profile.

Honest limitations · 2 / 4 — Reference-data quality

RRUFF is heterogeneous. Among 6 RRUFF specimens labelled Magnetite:

  • 4 have their dominant peak at 660–682 cm⁻¹ — correct
  • 2 have the dominant intensity at 100 cm⁻¹, the edge of the grid — instrumental cut-off artefact

Averaging across all 6 dilutes the genuine peak; the matcher picks Westerveldite for a held-out Magnetite query.

Down-stream impact

  • Matcher accuracy on Magnetite drops to ~77 %; CNN saves the day at 90 %.
  • The synthetic ref helps here but does not have low-frequency lattice modes, so it cannot cancel the bad samples.

Properly fixing this needs

  • A per-spectrum quality flag (signal-to-baseline ratio, peak count, edge-energy)
  • Median rather than mean averaging
  • Curator review — not in scope for a mini project

Honest limitations · 3 / 4 — Coverage and class space

  • Synthetic library = 25 minerals. Below the count of common rock-forming and ore phases needed for routine geological work (≈ 60–100 species). Every mineral outside this set has only a noisy RRUFF average to anchor against.
  • CNN class space = 851 minerals, not 2 481. The min_spectra ≥ 2 filter excludes 1 631 species with a single RRUFF specimen. They are still searchable by the spectral matcher, but the network cannot output them.
  • Mixture decomposition is restricted to the 25-mineral basis. A genuine field sample is unlikely to be a clean linear combination of those 25; triclinic feldspars, micas, and most clay minerals are absent.
  • Spectral aliasing: minerals with peaks within ±10 cm⁻¹ of a Quartz 464 line (Sodalite, Amicite, Trattnerite, …) consistently land in the runner-up positions. We do not currently use polarisation, anisotropy, or hardness metadata to disambiguate.

Honest limitations · 4 / 4 — Methodological gaps

ConcernStatusHonest assessment
Uncertainty quantification Top-1 softmax Not calibrated — temperature scaling not yet applied; no ECE measured.
Mixture additivity Linear NNLS Assumes intensities add linearly; fluorescence and self-absorption violate this.
Combined-metric weights 50 : 50 grid-searched Heuristic — a learned weighting per query would improve borderline cases.
RF model deployment Trained, ~92 % val acc Joblib pickle fails to load on memory-pressured Windows hosts (75 MB allocation).
Negative testing Not implemented The system always returns a top-K — no "this is not a mineral" rejection.
Single-spectrum context By design Real Raman maps have spatial context we discard.
Domain validation Internal, RRUFF only Never tested on a live spectrometer; literature compatibility only.

The most uncomfortable limitation

Stated plainly

The system looks highly accurate on its own demo set because the demo set is generated from the same peak tables that build the synthetic reference library.

A 99.8 % match on quartz.csv is therefore an upper bound on what the pipeline does under ideal conditions, not a field-realistic accuracy.

The held-out RRUFF panel (six minerals, six correct, scores 86–97 %) is the more representative number. We currently have no out-of-distribution test set drawn from a spectrometer not represented in RRUFF, and constructing one is the most important next step.

Future work (in priority order)

  1. Out-of-distribution test set. Acquire spectra on a different spectrometer; report top-1 / top-5 there as the headline number.
  2. Calibrated confidence. Temperature scaling on the CNN; report Expected Calibration Error.
  3. Curated synthetic library expansion from 25 → ~80 minerals covering the major rock-forming species.
  4. Quality-aware RRUFF aggregation — median, plus a per-spectrum cleanliness score gating inclusion in the average.
  5. Negative class. Rejection threshold on the combined-metric score below which the answer is "no confident match".
  6. Train CNN on the same preprocessing pipeline used at inference (currently the RRUFF cache was built with an earlier preprocessing).
  7. End-to-end pytest suite covering all six demo cases plus a regression panel of held-out RRUFF specimens.

Conclusion

  • A practical Flask + PyTorch + SciPy pipeline that turns a CSV of Raman intensities into a mineral identification with a mixture decomposition in under one second on commodity hardware.
  • Two complementary identifiers — a 1-D ResNet CNN and a combined-metric spectral matcher — are reconciled by a confidence-aware dispatcher that each picks the right tool for clean and noisy queries respectively.
  • A mineral-aware mixture decomposition based on candidate-set NNLS produces physically interpretable phase fractions instead of arbitrary sparse explanations.
  • The honest bound: held-out RRUFF top-1 in the 86–97 % range across six tested minerals; the demo-set 99.8 % is an upper bound on ideal-condition behaviour, not field accuracy.
  • The most consequential next step is an external test set drawn from a spectrometer not represented in RRUFF.
